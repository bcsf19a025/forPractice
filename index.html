Difference Between Git & Github

Let's start with Difference between these two terms.

  • Git is a version control system that lets you manage and keep track of your source code history
  • GitHub is a cloud-based hosting service that lets you manage Git repositories

First of all, you need to install Git on your system which allows you to run linux commands on your system unless you are using Linux.

Git allows you keep track of files you are adding.

First when we make some changes(added some file or anything), git will realize that you have made some changes, now we have stage our changes. This is the second step in using git and making commit. First you made some changes, now you will add those files to the staging area.

Run the command git add . on your Git Bash Terminal. What this command will do will add all those files to staging area. You can also mention the file name incase of "." in which you have made changes.

Now finally run the command git commit -m "some useful message" to commit your changes.
